# Changelog

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。存档兼容性:数据库带事务化自动迁移,旧存档打开即升级,无需手动操作。

## v0.2.0 — 2026-07-22

主题:**养不出满分小孩**。v0.1 里只要按时喂、及时哄,所有数值一起涨;本版让"认真养"也需要取舍,每种爱法都留下偏向。设计参照社区提案(养成取舍机制),全部机制纯本地计算,零新增 API 依赖。

### 新机制

- **消化负荷**(`child_state.digest_load`,schema v6):一天听太多话会"吃撑"——过载(≥70)后新语料训练权重与营养 ×0.5(语料照样入库,只是学得浅),说话变碎(温度升/句长缩/叠词回升);白天每小时消化 3 点,夜窗(23:00-07:00)每小时 8 点=睡一觉基本清空。偷学(archive)是被动输入,不进消化账。
- **同类动作当日收益递减**:当日第 n+1 次同类照料动作效果 ×0.75ⁿ(下限 0.25)。**夜哭窗口内的 feed/soothe/diaper 永远全额**;discipline/neglect/homecoming 等重事件不衰减。
- **情境化安抚**:孩子平静时(无夜哭窗且心情 ≥55)被哄=不安减免砍半+悄悄攒依赖(独立轴 -0.3,流水 reason=`calm_soothe`);真难受时全额不变。
- **家庭词块**(`chunks.py`,schema v7):每晚"睡眠整理"从他真实听过的话里提取高频短片段("抱抱""不要走"),幼儿期起按概率整词起头说话(infant 0 → toddler 0.35 → adult 0.1)。词块只是采样种子:字符必须全部在当前词汇解锁集内,护栏照跑,模型本体零改动。
- **场景标签**(`corpus_item.scene`):新语料按动作上下文自动打场景(夜哭窗→comfort/夜段→bedtime/teach→teaching/偷听→overheard/其余 daily),说话时按触发源优先抽当下场景的词块;旧语料 NULL=legacy,不回溯。
- **观察日志**(`observer.py`):每天 21 点后从**真实统计**派生至多 2 行旁观记录进事件流(`nursery.event`)——重复词/没说完整(护栏真重试过)/白天自己安静待了很久(窗内无哭闹实证)/新字 ≥5/滚动 72h 没说过的家常词。查不出数据支撑=不发,绝不编。
- **双照护人关系**(`bond.py`,schema v8):孩子对 papa/mama 的感情分开长,四维=亲近/安心/踏实/委屈(0-100,流水全留痕)。v1 两槽模型:mama 通道记 mama 账,其余照护人(含 NURSERY_PLAYERS 自定义 persona)记主账 papa。夜哭窗内响应给主照护人记安心+踏实(每夜一次,`parenting_meta` 占位去重);管教攒委屈;平静时也哄的那个人他更黏。老档升级用既往动作账**半额估底**,`bond_confidence=low` 明示是估算。心理层输入新增"对爸爸/对妈妈"趋势行(prompt 模板零改动,digest `input_rev=2`)。
- **成长画像**(`portrait.py` + `driver <persona> portrait`):全量纯派生 JSON——三轴趋势/双照护人四维/夜哭逐夜账/语料构成(来源·声部·场景)/语言面(词汇量·拒绝率·涌现率·top5 词块)/关键第一次清单。零写入,只给事实不打人格标签。
- **首次记录四连**:第一次说「不要」/第一次说自己的名字/第一次说出没人教过的组合/第一次整词说话,进成长相册(婴儿期乱语挡误触发)。

### 修正与口径

- `model.feed` 训练权重支持小数(下限 0.1)——过载打折真实作用到计数(此前取整到 ≥1 会把打折圆没);历史整数快照原样兼容。
- 心理轴流水改记**实际生效增量**(0-100 夹取后的 after-cur),饱和顶格=零增量不落行——趋势不再虚报(非饱和区间行为不变)。
- 夜哭"每夜一次"类加成的去重占位移到 `parenting_meta`(与流水解耦)——修复饱和夜标记丢失可重复领的问题。
- `utterance.rejection_reason` 分类修准:零重试的空模型=`no_model`,`guard_exhausted` 必有 retries>0 实证。
- 忽视账与画像统一逐夜口径 `events.closed_cry_nights`(只算已完结的 fired 主哭夜、窗内响应;combo 不摊分母);结局判分自 v0.1 起已按夜计,口径不变。
- 状态结算步长贴夜窗边界切开:消化分量为分段常速率的精确积分,任意读取节奏结果一致。

### 配置项(全部可调,见 `config.py` 对应注释)

`RULES_V2_SINCE`(默认 0=新档全程生效;运营老档可设未来时刻延后切换)、`DIGEST_*`、`DAILY_DECAY*`、`CALM_SOOTHE_*`、`CHUNK_*`、`SPEAK_SCENE_HINT`、`OBSERVE_*`、`BOND_*`。

### 兼容性

- 存档 v5→v8 自动迁移(新增列/表,不改既有数据)。老档迁移时会按孩子钉下 `rules_v2_since` 时刻戳:**升级前的动作/语料不进新规则计算**(不追溯是真承诺,升级当日的旧动作也不计入当日递减)。想让老档更晚才进入取舍规则,把 `RULES_V2_SINCE` 设为未来时刻即可(生效时刻取两者较大)。
- 测试:129 → 208(新增 79 项覆盖全部新机制与边界)。
