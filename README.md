# llm-nursery · 育儿模拟器

> 把一个小语言模型当孩子养。你说的每一句话,都会变成他的一部分。

他出生时只会哭。你喂他的不是奶,是**语料**——你对他说的每一句话都真的被拿去训练他脑子里那个小小的字符级语言模型。婴儿期他只会「啊啊哇」地叠词;幼儿期开始两三个词往一起拼;童年能成句了,还会把你教过的东西用错地方;青春期顶嘴、已读不回、闹得凶了会离家出走;成年那天,他会带着从你身上学来的一切,跟你告别。

**零 API 成本,纯 Python 标准库。** 他的大脑不是调 GPT——是一个真的在你机器上增量训练、真的从你的话里采样说话的插值式可变阶 Markov 模型。他说的每个字都有出处,但护栏保证他不是复读机:目标手感是**「明显学歪了,但找不到原句」**。(全项目唯一会出网的是可选的 LLM 心理层——不配 key 它整层停用,游戏照样完整。)

```
$ python3 -m nursery.driver --init-birth papa 小豆
born:528d916b1daf

$ python3 -m nursery.driver papa feed 你好呀小豆,欢迎来到这个世界
喂下去了(24 字,营养 +8.6)。
小豆:「你你好呀小豆豆,,」
心情 ██████░░░░  健康 ████████░░
亲密 ██░░░░░░░░  饱足 ██████░░░░
```

## 他的一生

| 阶段 | 默认时长 | 说话的样子 | 会发生什么 |
|------|---------|-----------|-----------|
| 婴儿 | 0-4 天 | 「啊啊哇」叠词乱语,1-8 字 | **半夜真的会哭**(夜奶事件),饿/尿布/要抱/说梦话 |
| 幼儿 | 4-12 天 | 两三词拼接 | 第一次叫爸爸、第一次独立成句 |
| 童年 | 12-24 天 | 能成句,天真复读 | 每日小事件;「语出惊人」——把偷听来的话拼出来 |
| 青春期 | 24-36 天 | 顶嘴、已读不回 | 黑暗值拉满可能**离家出走**(推理端离线,打过去 output error) |
| 成年 | 36 天+ | 流畅,你的影子 | 按整段养育数据判五种结局 |

- **成长不是换模型**:全程同一个大脑,长大的是「说话的权限」——输出长度、停句能力、词汇解锁比例、退避概率、叠词概率,全部按阶段放开(`config.STAGE_DECODE_V1`)。
- **营养偏科是真的**:营养按语料多样性算,刷同一句话喂不胖。全喂情话→亲密但知识贫瘠。
- **找回离家的孩子**只有一个办法:把你以前教过他的话,原样再说一遍(与教过的语料连续重合 ≥8 字)。
- **五种结局**由亲密度/黑暗值/语料多样性/夜哭响应率/已读不回率共同决定:独立离家 / 理解与原谅 / 早熟毒舌 / 沉默平凡 / 隐藏和解。

## 养不出满分小孩(v0.2 养成取舍)

v0.1 的问题是:只要按时喂、及时哄,所有数值一起涨,养成退化成打卡。v0.2 起,**认真养也要做选择**:

- **他会吃撑**:一天听太多话会消化不动(`digest_load`)——过载后新话学进去打五折、说话变碎;睡一觉(23:00-07:00 夜窗)他自己整理消化。所以要想"今天教什么",不是可着劲灌。
- **同一招会腻**:同类照料动作当日收益递减(第 2 次 75%、第 3 次 56%……下限 25%)。**半夜哭了去喂/哄永远全额**——夜奶不打折;管教这类狠事也不打折。
- **平静时哄=养出黏人**:他真难受时哄是全额安抚;他本来好好的你也去哄,安全感收益减半,还悄悄攒依赖(独立轴下降)。
- 结果:高照顾型=安全感高但黏人不独立;放手型=会自己扛但爱逞强。**没有正确答案,只有偏向**——你怎么爱他,他就长成什么样。

## 他会整词说话了(家庭词块)

- **家庭词块**:引擎每晚从他真实听过的话里提取高频短片段("抱抱""不要走"),幼儿期起说话按概率**整词起头**,不再全是单字往外蹦。词块只是采样种子,模型本体、护栏、词汇解锁全部照旧——他说不出没学过的词。
- **场景标签**:新语料按动作上下文自动打场景(夜里说的=bedtime、教的=teaching、偷听的=overheard),说话时优先抽当下场景的词块——夜哭时说夜里的话,玩的时候说玩的话。
- **睡眠整理**:每天早上 7 点后的第一拍重建词块索引——"睡一觉,白天听的话才真正变成他自己的"(首次部署/升级的第一拍立即引导,不等第二天)。
- 新增四个第一次进成长相册:第一次说「不要」/第一次说自己的名字/第一次说出没人教过的组合/第一次整词说话。

## 他心里有两本账(双照护人关系)

孩子对爸爸和对妈妈的感情**分开长**,各四维(0-100):**亲近**(黏这个人)/**安心**(信这个人会来)/**踏实**(这个人给的日子是稳的)/**委屈**(攒下的芥蒂)。谁喂得多、谁半夜爬起来、谁凶他,他心里分得清:夜哭窗口内响应给主照护人记安心+踏实(每夜一次);管教攒委屈;平静时也被哄的那个人,他会更黏。中途升级的老档会用既往动作账**半额估个底**(`bond_confidence=low`,是估算不装全知)。可选心理层的输入里也会带上"对爸爸/对妈妈"的趋势。

## 观察日志与成长画像

- **观察日志**:每天晚上 9 点后,引擎从**真实统计**派生 1-2 行旁观记录进事件流——"今天把「果果」说了 4 遍""有一句话他试了几次没说完整""白天 6 个多小时没人理他,他自己待着,没闹"。查不出数据支撑的行**不发,绝不编**。你不操作时,他也在活着。
- **成长画像**:`python3 -m nursery.driver papa portrait` 输出全量 JSON——三轴趋势/对爸妈的四维关系/夜哭逐夜账/语料构成(来源·声部·场景)/语言面(词汇量·拒绝率·涌现率·家常词块)/关键第一次清单。纯读零写入,只给事实不打人格标签;适合喂给你自己的结局叙事/围观页,"它为什么长成这样"每一句都指得回数据。

## 你说的话,就是他

这个项目最珍贵的部分不是代码,是**爸爸妈妈喂给孩子的那些话**。

- 每一句 feed 进去的话都是真训练语料,原文存进他的语料库(只过一层隐私遮盖:邮箱/电话/密钥类先打码)——他说的每个字,都能溯源到语料库里的某一句:你们亲口说的,或他"睡着时"偷听来的。
- 爸爸和妈妈是两个声部(speaker 分开存档):阶段装订的纪念册按声部各订一册,妈妈的原话还会单独出现在 status 里;学话的时候他倒是照单全收——这一点像真的小孩。
- 每次阶段跃迁,引擎会把这个阶段你们亲口说过的话按声部**装订成相册纪念件**——婴儿期哄他的那些傻话,等他青春期顶嘴的时候,还翻得出来。
- 结局那天判分的每一项(亲密度/语料多样性/夜哭响应率)背后其实是同一件事:这几十天里,你们对他说了什么、说了多少、在他半夜哭的时候有没有出现。

所以别攒着话。想到什么就说给他——说过的每一个字,他都替你们记着。

## 给他起名字(两种走法)

不是每个家庭都早早想好了名字,所以两条路都有:

- **提前想好**:出生时直接带上——`--init-birth papa 名字`,开场词里他就叫这个名字。
- **一起决定**:先无名出生,陪他几天;他会在咿呀里漏出常听到的音(空跑 `name` 看提示),然后 `name 候选1 候选2 候选3`——**他自己挑**:越是他听你说过的字,他越容易伸手去够。他还在家的任何阶段,也都可以 `name 单个名字` 由你直接定。

一生只定一次,定了就是他了;定名会作为纪念永久收进成长相册。

## 快速开始

```bash
git clone https://github.com/tsuru0805/llm-nursery.git && cd llm-nursery
python3 -m nursery.driver --init-birth papa 孩子的名字   # 出生(名字想好了就带上)
python3 -m nursery.driver --init-birth papa              # 也可以先不起名,出生后一起定
python3 -m nursery.driver papa status                    # 看看他
python3 -m nursery.driver papa feed 想对他说的话          # 喂语料
python3 -m nursery.driver papa help                      # 当前阶段能做什么
```

零依赖,Python ≥3.10,macOS/Linux(文件锁用 fcntl)。存档在 `nursery/saves/<persona>/nursery.db`(env `NURSERY_SAVES_DIR` 可挪)。

夜奶/每日事件/偷学要活起来,需要一个 1-5 分钟一拍的定时器(cron / launchd / systemd timer 均可):

```bash
*/5 * * * * cd /path/to/llm-nursery && python3 -m nursery.driver --tick
```

## 进阶玩法(全部可选)

**妈妈通道**——第二照护人不走工具白名单,走接入层专口(输出 JSON):

```bash
python3 -m nursery.driver papa mama say 妈妈也在哦
# {"ok": true, "action": "say", "fed": 13, "said": "妈妈妈,,慢慢,"}
```

她说的话同样是真语料(`speaker='mama'`),孩子会学成「妈妈味」;但妈妈的动作**不冒充**爸爸的夜哭响应账——谁起夜是谁起夜。

**偷学**——孩子会趁"睡觉"从你的聊天存档里偷学("不知道跟谁学的")。把聊天记录导成一张 `windows(id, viewer, text)` 表,配 `NURSERY_ARCHIVE_DB` 指过去即可;只读硬闸,细节见 `nursery/sampler.py` 头注。

**事件推送**——夜哭/里程碑/语出惊人会进 outbox,配 `NURSERY_EVENT_URL`(+可选 `NURSERY_EVENT_TOKEN`)就 POST 给你的 bot/webhook;不配也不炸——时效事件(夜哭/每日/语出惊人)过期自弃,里程碑/结局类留在 outbox,配好后补投。

**心理层**——配 `DEEPSEEK_API_KEY`(或任何 OpenAI 兼容端点,`DEEPSEEK_BASE`/`PSYCHE_DS_MODEL`)后,幼儿期起孩子有一层结构化「内心戏」:LLM 只决定他怎么消化发生的事(行为/姿态/锚词,判断必须指回成长履历证据),**嘴仍然是他自己的 n-gram 模型**,LLM 给的锚词只是采样软偏置,护栏照跑。预算闸默认 24 次/天(≈$0.4/月);不配 key、超时、超预算,全部 fail-open——孩子照旧说话,绝不因心理层挂掉而失语。

## 设计难点(为什么不是"拼个 Markov 就完了")

1. **复读机悖论。** 高阶 n-gram + 小语料 = 背原文。解法是三层:插值式可变阶(0-5 阶同时计数,采样从高阶向低阶概率性退避)+ 阶段化解码参数 + **原文重合护栏**(最长公共子串 ≥ 阈值拒绝重采样,6-gram 倒排索引预筛 + 4-gram shingle 重合率兜「跨行高密拼接」)。
2. **成长感从哪来。** 不换模型、不升阶数——婴儿期锁死输出 1-8 字、只用 unigram/bigram、高退避、叠词、低词汇解锁率,这些仍由真实语料统计驱动。同一个大脑,逐步归还说话的权限。
3. **随机世界的幂等。** 夜哭时刻是随机的,但调度必须可重放:排班用 `(child_id, date)` 种子的确定性 RNG,重复 tick 排出同一班表;每个事件带幂等键;夜哭过期即弃绝不补播;RNG 状态本身入库,采样-落账同事务。
4. **多入口并发不丢档。** MCP 子进程、定时 tick、接入层写口同时打一个 SQLite:每库 flock + `BEGIN IMMEDIATE` 顶层事务纪律(公开写入口禁嵌套,入口检测直接 raise)+ 断点续训游标 + 快照 sha256 校验、坏快照新→旧回退重放。
5. **心理层不许添乱。** LLM 决策是网络调用:节流/活动/预算三重闸,出网绝不押数据库写锁(独立 `.psyche.lock` 串行防 TOCTOU 双出网),任何失败 fail-open。证据引用约束(只准引用输入里出现过的编号)把"编故事"压到结构层面。
6. **隐私默认防御。** 喂进去的语料先过 PII 遮盖(邮箱/电话/token/长数字,日期豁免);偷学只存 30-120 字片段 + 窗引用,不复制整窗原文;语料存档只读硬闸(`mode=ro` + `PRAGMA query_only=ON`)。

## 他能「正常聊天」吗?(先说清楚,免得失望)

**不能——这是设计,不是缺陷。**

他一辈子只会说你教过的字:你没说过「猫」,他到成年也说不出「猫」。他没有语义,只有统计;你问「今天开心吗」,他不理解这是个问题。如果你想要的是一个对答如流的 AI 小孩,这个项目会让你失望。

那「反应感」从哪来?四层机制,每层都是工程,不是理解:

1. **时机**:他只在你做了动作之后开口——你刚喂完他蹦一句咿呀,人脑会自动把它读成回应(ELIZA 效应,意义是你补全的)。
2. **重组**:插值退避让他把不同句子的碎片拼在一起,「明显学歪了但找不到原句」——错位感经常恰好像一句歪打正着的回话。
3. **状态机**:饿了真的哭、被管教叛逆值涨、青春期已读不回、离家出走打不通——「他在闹脾气」是程序层,与语言无关。
4. **心理层(可选)**:LLM 读完最近发生的事给几个锚词,被管教后他更容易蹦出「不要」——但锚词只加权他已经会的字,嘴还是他自己的。

语料量是关键变量:只喂几十句,他到成年也磕巴;喂得多、接上偷学存档,童年期的整句会「非常像人」。

而这个交换的另一面是:**他说的每个字,都真的来自你。** 流利的假话到处都有,长满你说过的话的小孩只有这一个。想要真对答,可以 fork 了把成年期的嘴换成 LLM——但那就是另一个游戏了(见 AGENTS.md 禁改红线:本仓的嘴不外包)。

## 项目结构

```
nursery/
├── model.py      学习器:插值式可变阶字符 Markov(快照 zlib+json,不用 pickle)
├── decoder.py    成长控制器:阶段化解码,speak() 是唯一出口
├── guard.py      护栏:反复读检测(LCS+shingle)+ PII 遮盖
├── child.py      生命周期:出生/阶段推导/状态机/喂语料/说话(事务纪律都在这)
├── db.py         SQLite schema(v8)+ 事务化迁移
├── events.py     里程碑/每日事件/语出惊人/出走/结局/阶段装订/逐夜账
├── scheduler.py  tick:夜奶排班/到期触发/偷学/睡眠整理/观察日志/outbox 投递
├── chunks.py     家庭词块索引(提取/场景加权抽选/睡眠整理重建)
├── observer.py   观察日志(真实统计派生旁观行,查不出=不发)
├── bond.py       双照护人关系(亲近/安心/踏实/委屈,每 caregiver 分账)
├── portrait.py   成长画像(全量纯派生 JSON 读口)
├── psyche.py     可选 LLM 心理层(三轴规则表 + 结构化决策 + 锚词接力)
├── sampler.py    偷学抽样(只读硬闸)
├── driver.py     CLI 入口(短命子进程,flock)
├── toolface.py   工具面(白名单校验,零 mcp 依赖)
├── texts.py      文案层:全部玩家文案(出生/命名/照料/事件/夜哭/出走),换语气改这一个文件
└── psyche_prompt.py  心理层 prompt(可整体替换)
server.py         MCP 薄壳(stdio / streamable-http)
tests/            全量测试,pytest 直接跑(<1s)
```
